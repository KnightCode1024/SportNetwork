# Архитектура проекта

Проект построен на принципах **Clean Architecture** с явным разделением ответственности между слоями.

## Структура слоёв

```
┌─────────────────────────────────────────────────────────┐
│                    Controllers Layer                     │
│  (FastAPI routers, Pydantic schemas, HTTP adapters)     │
│  src/controllers/                                       │
└────────────────────────┬────────────────────────────────┘
                         │ зависит от
┌────────────────────────▼────────────────────────────────┐
│                  Application Layer                       │
│  (Interactors, Use Cases, Interfaces, DTOs)             │
│  src/application/                                       │
├─────────────────────────────────────────────────────────┤
│  • Interactors - бизнес-юзы (регистрация, логин, и т.д.)│
│  • Interfaces - Protocols для gateways и сервисов       │
│  • DTO - объекты передачи данных между слоями           │
└────────────────────────┬────────────────────────────────┘
                         │ зависит от
┌────────────────────────▼────────────────────────────────┐
│                     Domain Layer                         │
│  (Domain Models, Business Rules, Entities)              │
│  src/domain/                                            │
├─────────────────────────────────────────────────────────┤
│  • User - модель пользователя с логикой                 │
│  • Profile - модель профиля с валидацией                │
└─────────────────────────────────────────────────────────┘
          ▲                              ▲
          │                              │
          │ реализует                    │ использует
┌─────────┴──────────────┐  ┌───────────┴────────────────┐
│   Infrastructure Layer │  │      IoC Container         │
│  (Gateways, Services,  │  │   (Dishka DI providers)    │
│    DB, S3, Redis, JWT) │  │                            │
│  src/infrastructure/   │  │  src/ioc/                  │
└────────────────────────┘  └────────────────────────────┘
```

## Правило зависимостей

```
Controllers → Application → Domain
                                    ↑
Infrastructure ─────────────────────┘
     IoC ────────────────────────────┘
```

**Внешние слои зависят от внутренних, а не наоборот.**
